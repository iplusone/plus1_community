# ポータルサイト要件定義

## 1. 目的

本システムは、企業およびその配下組織（拠点）がそれぞれ独立した Web ページ（拠点(スポット)）を持ち、

- 情報発信
- 検索
- 集客

を行うためのポータル基盤である。

## 2. 基本概念

### 拠点(スポット)

- 拠点(スポット) = 組織単位（本社・支社・営業所・教室・店舗など）
- 1 拠点(スポット) = 1 ページ
- 最大 5 階層の親子構造
- 上位・下位で機能差なし（全拠点同一構造）

## 3. 管理権限

- 各拠点(スポット)は複数管理者を持つ
- 権限は階層に従う

| 管理者 | 管理範囲 |
| --- | --- |
| 上位 | 配下すべて |
| 中間 | 自組織 + 配下 |
| 下位 | 自組織のみ |

## 3.1 企業登録・承認フロー

### 初回企業登録

- 企業はメールアドレスを登録する
- システムは登録用 URL をメール送信する
- 企業は登録画面で企業情報と初回管理者情報を入力する
- 登録内容は審査待ちとなる
- 運営側が確認し、承認後に利用可能となる

### 承認後の状態

- 初回登録したユーザは、その企業の最上位管理者となる
- 最上位管理者は、自社配下の全拠点(スポット)を管理できる
- 最上位管理者は、配下拠点(スポット)を新規作成できる
- 最上位管理者は、配下拠点(スポット)ごとの管理者ユーザを招待できる

### 拠点管理者の登録

- 拠点管理者はメールアドレスで招待する
- 招待されたユーザには登録用 URL をメール送信する
- 招待ユーザは登録完了後、割り当てられた拠点(スポット)を管理できる

### 利用制御

- 運営承認前の企業は管理画面を利用できない
- 運営承認前の企業配下データは公開できない
- 招待済みでも登録未完了ユーザは管理操作できない

### 状態

- 企業: 仮登録 / メール確認済み / 審査待ち / 承認済み / 却下
- 招待ユーザ: 招待中 / 登録完了 / 失効

## 4. 拠点ページ機能

### 基本情報

- 拠点名
- 住所
- 電話番号
- 営業時間
- 定休日

### アクセス

- 地図
- 行き方

### 説明

- 概要
- 特徴

### サービス

- 提供内容
- メニュー
- 料金

### メディア

- 画像（複数）
- 動画

### スタッフ

- 名前
- プロフィール

### クーポン

- 内容
- 条件
- 有効期限

## 5. コンテンツ（WordPress）

- 拠点(スポット)単位で WordPress を持つ（任意・最大 1 つ）
- 記事は WordPress で管理

### カテゴリ

- ニュース
- ブログ

## 6. 公開制御

- 公開 / 非公開
- 公開日時（予約公開可能）

### 表示条件

公開状態 = 公開  
かつ  
公開日時 <= 現在

### 非公開

- 一般ユーザー閲覧不可
- URL 直接アクセス不可
- 検索対象外

## 7. 検索機能

### 地域

- 都道府県 -> 市区町村 -> 町名（3 階層）

### ジャンル

- 最大 3 階層
- 1 拠点(スポット)につき最大 3 件

### タグ

- 自由入力
- サジェストあり
- 非階層

### 検索条件

- 地域 x ジャンル x タグ

## 8. 検索 UI

- 地域: テキスト入力
- ジャンル: 入力 + サジェスト
- タグ: 入力 + サジェスト
- 実行: 検索ボタン

## 9. 検索結果

### 表示形式

- リスト / カード切替

### 表示項目

- サムネイル
- 拠点名
- 住所

### 件数

- 1 ページ 50 件

### ページング

- ページネーション

### 並び

- 新着（公開日時）
- 人気（PV）

## 10. トップページ

### 要素

- おすすめ（手動）
- 最新（公開日時順）
- ランダム（毎回ランダム）

### 件数

- 各 10 件

### 導線

- タップで詳細
- 「もっと見る」で一覧

## 11. レイアウト

- 全拠点共通
- 将来テンプレート選択可能

## 12. ライフサイクル

- 作成（非公開）
- 公開（即時 / 予約）
- 更新（即時反映）
- 非公開（即時非表示）

## 13. パフォーマンス要件

### 基本

- すべての操作が高速
- データ増加でも性能劣化しない

### 方針

- 検索専用データを使用
- JOIN を行わない
- 非正規化構造を許容

## 14. 検索データ構造

### 方針

- 管理データと検索データを分離

### 検索用データ

- 必要最小限の項目のみ保持

### 含む項目

- `spot_id`
- 拠点名
- 住所
- 地域情報
- ジャンル
- タグ
- 公開状態
- 公開日時
- `view_count`
- サムネイル

### 特徴

- 単一レコードで完結
- JOIN 不要
- 高速検索可能

## 15. 本質定義

本システムは

- 組織構造（拠点階層）
- コンテンツ（WordPress）
- 検索（地域・ジャンル・タグ）

を統合した

「拠点(スポット)ベースの高速ポータルプラットフォーム」である。
